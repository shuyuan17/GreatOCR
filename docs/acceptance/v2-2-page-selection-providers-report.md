# GreatOCR V2.2 页面选择与多模型架构验收报告

日期：2026-07-01  
分支：`codex/v2-2-page-selection-providers`

## 阶段结论

V2.2 已完成离线自动化验收。本阶段建立了页面选择、仅发送选中页面、供应商能力卡、通用多模态实验适配器、可选文本校正、失败切换策略、页面返工和结果版本机制。

## 已验收能力

- 支持 `3, 8-10, 9, 1` 等页码表达式，并校验空值、倒序、越界和非法字符。
- 生成只包含选中页面的临时子 PDF；保存任务页码到原 PDF 页码的映射。
- 中断恢复后可以从任务清单恢复页码映射。
- 默认合并选中页；按区间输出时生成 `result-pages-<范围>.docx`。
- 供应商注册表按文字、坐标、版面、表格、图片和公式能力筛选方案。
- 通用多模态实验适配器严格校验 JSON Schema、页码、置信度和归一化坐标。
- 智能文本校正默认关闭；失败时继续生成 Word；关键字段自动修改默认拒绝。
- 失败切换默认要求用户选择；自动模式只使用任务开始前批准且能力满足要求的方案。
- 任务清单记录批准的 profile ID 和确认时间，不保存 API Key。
- 页面返工只发送指定失败页；成功后替换对应页并保留其他页面稳定 ID。
- 首次结果保存为 `result-v1.docx`；返工继续生成 `result-v2.docx` 等版本，同时更新 `result.docx`。
- 新 DOCX 通过包结构验证后才发布为当前结果。

## 安全与外部服务说明

本阶段没有执行真实 MinerU 上传，没有调用千问、Kimi、豆包、DeepSeek 或其他线上模型，没有读取项目中的 API Key 文件，也没有向 GitHub 上传代码。
